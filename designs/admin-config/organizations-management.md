## Entity Definitions

### Region

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| Code | String | Yes | Unique identifier (e.g., "REG-001") |
| Name | String | Yes | Display name |
| Description | String | No | Optional description |
| Is Active | Boolean | Yes | Active/inactive status |
| Source | Enum | Read-only | LOCAL or HUB |

### District

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| Code | String | Yes | Unique identifier (e.g., "DST-001") |
| Name | String | Yes | Display name |
| Region Code | String | Yes | Parent region reference |
| Description | String | No | Optional description |
| Is Active | Boolean | Yes | Active/inactive status |
| Source | Enum | Read-only | LOCAL or HUB |

### Facility

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| Code | String | Yes | Unique identifier (e.g., "FAC-001") |
| Name | String | Yes | Display name |
| District Code | String | Yes | Parent district reference |
| Facility Type | String | Yes | Type of facility (Hospital, Clinic, Health Center, etc.) |
| Lab Type | String | No | Type of laboratory if applicable |
| Is Referral Lab | Boolean | No | Whether this is a referral laboratory |
| Operational Partner | String | No | Supporting organization/NGO |
| Address | String | No | Physical address |
| Contact Phone | String | No | Contact number |
| Contact Email | String | No | Contact email |
| Is Active | Boolean | Yes | Active/inactive status |
| Source | Enum | Read-only | LOCAL or HUB |

### Ward/Department/Unit

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| Code | String | Yes | Unique identifier (e.g., "WRD-001") |
| Name | String | Yes | Display name |
| Facility Code | String | Yes | Parent facility reference |
| Unit Type | String | No | Ward, Department, Unit, etc. |
| Is Active | Boolean | Yes | Active/inactive status |
| Source | Enum | Read-only | LOCAL or HUB |

- Source indicator (Local/Hub badge)
- Active status
- Actions (Edit, View Children, Deactivate)

- Items per page selector
- Page navigation

### Export

**Export Dropdown Options:**
1. Export All (complete workbook with all sheets)
2. Export Current View (filtered data only)
3. Download Template (empty template for import)

**Export Format:**
Excel workbook (.xlsx) with multiple sheets:
- Sheet 1: Regions
- Sheet 2: Districts
- Sheet 3: Facilities
- Sheet 4: Wards_Units

Each sheet includes:
- Header row with column names
- All relevant fields for that entity type
- Instructions row (hidden by default)

## Migration Considerations

- Existing organization data must be preserved
- Map current data structure to new hierarchy
- Assign "LOCAL" source to all existing data
- Generate codes for any entities missing them
- Validate referential integrity after migration

## Acceptance Criteria

### Core Functionality
- [ ] Sidebar navigation with entity type counts
- [ ] List view with search, filter, pagination
- [ ] Tree view for hierarchy visualization
- [ ] Source indicator (Local/Hub) on all entries
- [ ] Drill-down navigation with breadcrumbs
- [ ] Add/Edit forms for each entity type

### Import/Export
- [ ] Export all data as Excel workbook
- [ ] Export filtered data
- [ ] Download empty template
- [ ] Import with file upload
- [ ] Validation preview showing new/updated/errors
- [ ] Row-by-row change review
- [ ] Field-level diff for updates
- [ ] Confirmation before import
- [ ] Error report download

### Data Integrity
- [ ] Enforce one-to-one relationships
- [ ] Validate parent references on import
- [ ] Prevent deletion of parents with children
- [ ] Hub data restrictions enforced
